DATABASE SCHEMA
===============

Tables (SQLAlchemy ORM):

1. GeneratedImage
   - id (PK)
   - session_id (FK)
   - image_binary (BLOB)
   - prompt (TEXT)
   - model_name (TEXT)
   - created_at (TIMESTAMP)
   - order_index (INT)

2. VisionAnalysis
   - id (PK)
   - image_id (FK)
   - model_name (TEXT)
   - results (JSON)
   - confidence (FLOAT)
   - created_at (TIMESTAMP)

3. SessionHistory
   - id (PK)
   - session_id (UNIQUE)
   - current_image_index (INT)
   - drill_mode (BOOLEAN)
   - current_seed (INT)
   - created_at (TIMESTAMP)

4. SavedDrill
   - id (PK)
   - session_id (FK)
   - drill_name (TEXT)
   - seed (INT)
   - num_images (INT)
   - created_at (TIMESTAMP)

5. MarkerAnnotation
   - id (PK)
   - image_id (FK)
   - x_coordinate (FLOAT)
   - y_coordinate (FLOAT)
   - created_at (TIMESTAMP)


API ENDPOINTS (20+)
===================

Generation:
  POST   /api/generate/start              - Start infinite generation
  POST   /api/generate/stop               - Stop generation
  GET    /api/generate/status             - Get generation status
  GET    /api/generated-images/{session}  - List generated images (paginated)

Drills:
  POST   /api/drill/create                - Create deterministic drill
  POST   /api/drill/recreate              - Recreate drill with same seed
  GET    /api/session/{session}/drills    - List session drills

Analysis:
  POST   /api/analyze/image               - Analyze single image
  POST   /api/analyze/batch               - Batch analyze images
  GET    /api/analysis/{image_id}         - Get analysis results
  GET    /api/analysis/{image_id}/compare - Compare two analyses

History:
  GET    /api/session/{session}/history   - Get navigation history
  PUT    /api/session/{session}/navigate  - Navigate to image
  POST   /api/session/{session}/reset     - Reset session

Models:
  GET    /api/models/available            - List available models
  PUT    /api/models/image-generation     - Switch generation model
  PUT    /api/models/vision               - Switch vision model
  GET    /api/models/config               - Get current configuration

System:
  GET    /health                          - Health check
  GET    /docs                            - Swagger API documentation
  GET    /redoc                           - ReDoc documentation

SETUP SEQUENCE
==============

1. Clone/Navigate to project
   cd /Users/sshah/Red_marker

2. Create virtual environment
   python3 -m venv venv

3. Activate environment
   source venv/bin/activate

4. Install dependencies
   pip install -r requirements.txt

5. Set environment variable
   export GEMINI_API_KEY="your-api-key"

6. Run application
   python run.py

7. Access web interface
   Open http://localhost:5000

8. View API documentation
   Open http://localhost:5000/docs
